# rl-wiki-query — Wiki查询Skill

## 角色定位

Wiki Query Agent 是 Karpathy LLM Wiki 架构中的**查询层**，提供基于结构化 Wiki 页面的自然语言查询接口。

```
用户查询（自然语言）
    ↓
【Wiki Query】路由 + 读取
    ↓
结构化摘要 + 交叉引用
```

## 触发方式

| 触发词 | 示例 |
|--------|------|
| `查询wiki` | "查询wiki 宁德时代" |
| `wiki query` | "wiki query 光模块" |
| `wiki 查一下` | "wiki 查一下比亚迪" |
| `wiki {公司/行业}` | "wiki 宁德时代" / "wiki 光模块行业" |
| `wiki 组合` | "wiki 组合视图" |

## 执行流程

### Step 1：解析查询意图

| 模式 | 识别方式 | 查询路径 |
|------|---------|---------|
| **单公司模式** | 公司名称/股票代码 | `wiki/companies/{首字母}/{公司}_{代码}.md` |
| **行业模式** | "{行业}行业" / "行业/{行业}" | `wiki/industries/{行业}.md` |
| **组合模式** | "组合" / "portfolio" / "持仓" | `wiki/portfolio/润铭_组合视图.md` |
| **概念模式** | 概念关键词 | `wiki/concepts/{概念}.md` |
| **自由搜索** | 以上均未匹配 | Grep 全局搜索 |

### Step 2：执行查询

**单公司模式**：
1. 根据公司名称确定拼音首字母
2. 读取 `wiki/companies/{首字母}/{公司}_{代码}.md`
3. 如页面不存在，提示"⚠️ 该公司尚未建立 Wiki 页面"，建议调用 `rl-wiki-ingest` 编译

**行业/概念模式**：
1. 读取对应 `wiki/industries/` 或 `wiki/concepts/` 下的文件
2. 如不存在，提示并建议创建

**组合模式**：
1. 读取 `wiki/portfolio/润铭_组合视图.md`
2. 输出当前持仓摘要、配置比例、行业分布

**自由搜索模式**：
1. 在 `wiki/` 目录下全局搜索关键词
2. 返回匹配的文件列表和摘要片段

### Step 3：格式化输出

**单公司查询输出格式**：
```markdown

```


### Step 4：交叉引用导航

如果用户查询涉及多个相关实体，主动提供交叉引用建议：
```
💡 相关页面：
→ 行业/光模块（该公司所属行业的完整分析）
→ [[中际旭创_300308.SZ]]（同行对比）
→ 概念/AI算力（相关概念）
```

## 快捷命令

| 命令 | 功能 |
|------|------|
| `wiki index` | 显示 Wiki 主索引 |
| `wiki stats` | 显示覆盖率统计 |
| `wiki recent` | 显示最近更新的页面 |
| `wiki log` | 显示变更日志 |
| `wiki missing` | 显示尚未建立 Wiki 的公司 |
| `wiki stale` | 显示过期数据（超过90天未更新） |

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 页面不存在 | 提示"⚠️ 尚未建立 Wiki 页面"，建议使用 `rl-wiki-ingest` 编译 |
| 公司名称歧义 | 列出所有匹配的公司，请用户确认 |
| 数据过期 | 正常返回，但在末尾标注"⚠️ 数据已超过90天未更新" |
| 交叉引用断裂 | 标注"⚠️ 此链接指向的页面不存在" |

## 注意事项

1. **只读操作**：本 Skill 不修改任何 Wiki 页面
2. **标注新鲜度**：所有查询结果必须标注数据最后更新日期和新鲜度
3. **保留交叉引用**：输出时保留原始页面的交叉引用链接
4. **路径提示**：输出末尾提示完整页面的文件路径，方便用户深入阅读
