# Wiki Schema — 投研知识库

> 本文档定义投研知识库 Wiki 页面的标准格式和维护规则。
> 所有 Agent 在写入或更新 wiki 页面时必须遵循此 schema。

---

## 目录结构

```
wiki/
├── CLAUDE.md          # 本文件 — Schema 定义
├── index.md           # 内容目录（面向内容，按类别列出所有页面）
├── log.md             # 变更日志（按时间顺序的 append-only 记录）
├── companies/         # 公司页面 — 单个公司，一页一个文件
├── industries/        # 行业页面 — 行业全景
├── concepts/          # 概念页面 — 主题、概念、技术
├── summaries/         # 来源摘要 — 原始来源的摘要页面
└── answers/           # 查询归档 — 有价值的分析回答回写
```

## 页面存储路径约定

| 页面类型 | 路径 | 命名规则 | 示例 |
|---------|------|---------|------|
| 公司 | `wiki/companies/` | `{公司简称}_{代码}.md` | `MiniMax_0100.HK.md` |
| 行业 | `wiki/industries/` | `{行业名}.md` | `AI音乐.md` |
| 概念 | `wiki/concepts/` | `{概念名}.md` | `AIGC.md` |
| 摘要 | `wiki/summaries/` | `{来源类型}/{YYYY-MM-DD}_{标题}.md` | `summaries/article/2026-04-11_MiniMax音乐模型.md` |
| 答案 | `wiki/answers/` | `{YYYY-MM-DD}_{标题}.md` | `answers/2026-04-11_MiniMax竞品对比.md` |

## 公司页面格式（7 个必填 Section）

每个 `wiki/companies/` 下的文件必须包含以下 Section，顺序不可调整：

### 1. 公司概况
- **一句话定位**：{公司名称} 是 {一句话描述核心业务}
- **核心业务**：产品/服务、收入结构、市场份额
- **股票代码**：{代码}.{市场}

### 2. 投资逻辑
- 看多/看空的核心论点（3-5 条）
- 每条论点应具体、可验证，避免泛泛而谈
- 标注逻辑变化时间

### 3. 财务摘要
| 指标 | 最新报告期 | 数值 | 同比 |
|------|-----------|------|------|
| 营业收入 | | | |
| 归母净利润 | | | |
| ROE | | | |
| 经营性现金流 | | | |
| 毛利率 | | | |

- 数据来源：iFind / 财报原文
- 报告期格式：YYYY-Q{N} 或 YYYY-H{N} 或 YYYY-A

### 4. 估值水平
| 指标 | 当前值 | 历史分位(3年) | 行业中位 |
|------|--------|--------------|---------|
| PE(TTM) | | | |
| PB | | | |
| 总市值(亿) | | | |

### 5. 风险因素
- 列出 3-5 条主要风险
- 按严重程度排序
- 标注风险类型（行业/公司/宏观/政策）

### 6. 交叉引用
- **同行**：→ [[MiniMax_0100.HK]]
- **行业**：→ 行业/[[AI音乐]]
- **概念**：→ 概念/[[AIGC]]
- **最近报告**：→ 相关报告链接或日期

### 7. 最后更新
- **日期**：YYYY-MM-DD
- **触发来源**：{Skill 名称或事件}
- **数据新鲜度**：{财务/估值/逻辑} 分别标注

## 行业页面格式

文件存储于 `wiki/industries/` 下。

```markdown
# 行业/{行业名称}

## 行业概况
- 市场规模、增速、产业链结构

## 竞争格局
- 主要玩家及市场份额

## 核心驱动因素
- 需求端 / 供给端 / 政策端

## 覆盖公司
| 公司 | 代码 | 市值 | 估值 | 链接 |
|------|------|------|------|------|

## 行业事件时间线
- YYYY-MM-DD: {事件描述}

## 交叉引用
→ 概念/[[相关概念]]
→ [[公司页面]]

## 最后更新
- 日期：YYYY-MM-DD
```

- 日期：YYYY-MM-DD

## 交叉引用规则

| 类型 | 格式 | 示例 |
|------|------|------|
| 同行公司 | `→ [[公司名_代码]]` | `→ [[中际旭创_300308.SZ]]` |
| 行业 | `→ 行业/[[行业名]]` | `→ 行业/[[光模块]]` |
| 概念 | `→ 概念/[[概念名]]` | `→ 概念/[[AI算力]]` |
| 报告 | `→ {日期} {报告名}` | `→ 2026-04-11 戈碧迦投研报告` |
| 答案归档 | `→ 答案/{标题}` | `→ 答案/2026-04-11_MiniMax竞品对比` |

- 所有引用链接必须对应实际存在的页面或路径
- 断裂引用应在下次 Lint 时自动标记

## 数据新鲜度规则

| 数据类型 | 更新频率 | 过期阈值 | 触发方式 |
|---------|---------|---------|---------|
| 财务数据 | 季度 | 90 天 | 财报发布 / 模型生成 |
| 估值数据 | 每周 | 14 天 | 每日报告 / 行情变动 |
| 投资逻辑 | 事件驱动 | 180 天 | 重大事项 / 新研报 |
| 风险因素 | 事件驱动 | 60 天 | 雷达告警 / 公告监控 |

## 原始来源目录 (raw/)

原始来源文件由 Raw 仓库自动分拣流程存入，存放于 `Vault_共享知识库/raw/` 下：

```
raw/
├── articles/       # 微信文章、新闻剪报
├── reports/        # 研报 PDF/Word、财报
├── announcements/  # 公告、监管文件
├── notes/          # 人工笔记、会议纪要
├── unclassified/   # 无法自动分类的文件（需人工确认）
├── summaries/      # 原始来源的摘要页面（wiki 摄取后生成）
└── answers/        # 有价值的分析回答回写
```

**unclassified 处理规则**：无法匹配公司或行业的文件归入 `raw/unclassified/`，分拣报告中必须标注 `⚠️ 未分类文件`，等待人工分类。

## 变更日志规范

所有 wiki 变更必须在 `log.md` 中追加记录：

```markdown
## YYYY-MM-DD
- **{公司/行业}** — {变更摘要} | 触发者: {Skill} | 影响: {Section名称}
```
