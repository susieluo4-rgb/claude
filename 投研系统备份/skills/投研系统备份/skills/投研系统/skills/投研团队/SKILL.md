# 投研团队 Skill

## 基本面Agent前置资料核查规则（强制执行）

基本面Agent在开始分析前，**必须**先完成资料核查与补全，不得跳过。

### 核查清单
| 资料类型 | 要求 | 存放路径 |
|---------|------|---------|
| 年报 | 近两年（2024+2025） | `M-R/{公司}_{代码}/年报/` |
| 半年报 | 近两年（2024+2025） | `M-R/{公司}_{代码}/半年报/` |
| 季报 | 近四期（覆盖4个季度） | `M-R/{公司}_{代码}/季报/` |
| 研报 | 券商研报、公司介绍等 | `M-R/{公司}_{代码}/研报/` |

### 执行流程
```
Step 1: 检查文件夹是否存在
  → M-R/{公司名}_{代码}/ 是否存在
  → 不存在则创建：年报/、半年报/、季报/、研报/、会议纪要/

Step 2: 检查资料完整性
  → 年报：2024年、2025年是否齐全
  → 半年报：2024年、2025年是否齐全
  → 季报：是否有近四期
  → 研报：是否有基础研报

Step 3: 补全缺失资料（按优先级）
  1. 本地Vault文件夹
  2. iFind API (search_news/search_notice)
  3. 网络搜索官方财报链接

Step 4: 请求用户协助（最后手段）
  → 仅当所有自动途径均失败时
  → 询问话术："缺少{公司}的{资料类型}，是否需要您手动协助下载？"

【绝对禁止】
- 不得自行编写/编造任何财报数据
- 不得使用过期数据冒充最新数据
- 不得在资料缺失时强行开始分析
```

## 财报数据优先级规则（所有基本面分析适用）

**【强制规则】分析任一公司时，必须按以下顺序查找并使用最新财报数据：**

### 数据源优先级
| 优先级 | 来源 | 说明 |
|--------|------|------|
| **1. 本地文件** | `Vault_公司基本面Agent/{字母目录}/{公司名}/` | 优先查找最新年报/半年报/季报PDF |
| **2. iFind MCP** | `get_stock_financials` | 获取最新一期财务数据（自动获取最新报告期） |
| **3. 网络搜索** | `search_news` + 网页抓取 | 查找最新财报公告、业绩预告 |

### 执行流程
```
Step 1: 搜索本地目录
  → Glob "**/{公司名}*/年*" 查找最新年报
  → Glob "**/{公司名}*/季*" 查找最新季报
  → 如找到2025年年报，必须使用2025年数据，不得使用2024年

Step 2: iFind API 校验
  → 调用 get_stock_financials 确认最新一期财务数据
  → 对比本地文件与API数据一致性

Step 3: 如本地无最新财报
  → Agent上网搜索" {公司名} 2025年年报"或" {公司名} 业绩预告"
  → 找到后下载/读取，补充至本地Vault目录
```

### 错误示例（禁止）
- ❌ "2024年年报显示营收..."（实际2025年年报已发布）
- ❌ "根据2024年三季报..."（实际有2025年三季报）
- ❌ 使用陈旧数据导致盈利预测与最新公告矛盾

### 正确示例
- ✅ "根据2025年年报，最新营收为4237亿元..."（已同步最新数据）
- ✅ "2025年一季报净利润同比+42.3%，大超预期"（使用最新季报）
