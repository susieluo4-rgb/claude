# 四年级数学学习Agent - 网页版实施计划

## Context

用户希望将现有的命令行版数学学习Agent改成网页版，要求：
- 界面更友好
- 符合小学生（9-10岁）的喜好
- 支持动画、互动

## 目标

搭建一个彩色、卡通风格的网页版数学学习系统，包含：
1. 友好的用户界面（前端）
2. API服务（后端）
3. 与现有Agent系统集成

## 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | HTML/CSS/JS | 轻量，无需构建 |
| 后端 | FastAPI | Python现代Web框架 |
| 实时通信 | WebSocket / SSE | 实现打字机效果 |
| 界面风格 | 卡通+鲜艳色彩 | 适合小学生 |

## 设计风格

### 配色方案（明亮活泼）
- 主色：#FF6B6B（珊瑚红）
- 辅色：#4ECDC4（薄荷绿）
- 强调色：#FFE66D（阳光黄）
- 背景：#F7F7F7（浅灰白）
- 文字：#2D3436（深灰）

### 字体
- 主字体：思源黑体 / 幼圆
- 数字字体：大号、加粗、圆润

### UI元素
- 大按钮（便于点击）
- 圆角卡片
- 可爱的图标（emoji或SVG）
- 动画过渡效果

## 页面结构

```
index.html
├── 顶部导航栏（Logo + 学生名 + 学习统计）
├── 左侧边栏（功能菜单）
```

## 核心页面

### 1. 首页/仪表盘
- 今日学习目标
- 学习进度（环形进度条）
- 薄弱点提醒
- 快速开始按钮

### 2. 对话式学习界面
- 聊天气泡展示对话
- AI回复带打字机效果
- 题目卡片展示
- 选择题按钮点击交互
- 动画/图片展示区

### 3. 预习页面
- 主题选择卡片（可点击）
- 动画展示区
- 知识点卡片
- 小测验交互

### 4. 练习页面
- 题目展示区
- 答案输入/选择区
- 即时反馈动画
- 进度条

### 5. 错题集页面
- 错题列表卡片
- 错因分析展示
- 同类题练习

### 6. 学习报告页面
- 周报/月报图表
- 掌握度雷达图
- 成就徽章展示

## 实施步骤

### Step 1: 创建项目结构
```
Susie学习伙伴/
├── web/
│   ├── index.html          # 主页面
│   ├── css/
│   │   └── style.css       # 样式
│   ├── js/
│   │   ├── app.js          # 主逻辑
│   │   ├── api.js          # API调用
│   │   └── components.js   # UI组件
│   └── assets/
│       └── icons/          # 图标资源
├── api/
│   ├── main.py             # FastAPI入口
│   ├── agent_api.py        # Agent接口
│   └── models.py           # 数据模型
└── orchestrator/           # 现有系统（复用）
    └── ...
```

### Step 2: 实现后端API (FastAPI)
```python
# 主要API端点
POST /api/chat              # 发送消息，获取AI回复
GET  /api/progress/{student_id}  # 获取学习进度
GET  /api/errors/{student_id}    # 获取错题列表
POST /api/practice/{student_id}  # 提交练习答案
GET  /api/topics                 # 获取可选知识点
```

### Step 3: 实现前端页面
- 主HTML结构
- CSS样式（卡通风格）
- JavaScript交互逻辑
- 与后端API对接

### Step 4: 集成现有Agent
- 复用orchestrator
- 适配API接口
- 保持现有业务逻辑

## 关键文件

### 前端
- `web/index.html` - 主页面
- `web/css/style.css` - 样式
- `web/js/app.js` - 主逻辑

### 后端
- `api/main.py` - FastAPI应用
- `api/agent_api.py` - Agent接口封装

### 复用
- `orchestrator/` - 现有Agent系统
- `skills/` - 现有Skill
- `data/` - 现有数据模型

## 验证方式

1. 启动后端服务
2. 在浏览器打开index.html
3. 测试各功能流程：
   - 首页加载
   - 发送消息/AI回复
   - 预习流程
   - 练习答题
   - 错题查看
