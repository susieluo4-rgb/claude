# 纪要追踪分析 Skill

## 触发方式

**两步确认流程（无需指定公司）：**
1. 用户说 `纪要更新`
2. 用户粘贴纪要内容（公司名需包含在内容中）

**触发关键词：**
- `纪要更新`
- `纪要对比`
- `追踪纪要`

---

## 执行流程

### Step 1：用户粘贴纪要

```
📋 请粘贴会议纪要内容
   （公司名称需包含在正文内，将自动识别）
```

**纪要格式支持：** PDF、DOCX、TXT、MD、纯文本

---

### Step 2：提取公司名 + 确认股票代码

```
✅ 已识别公司：{公司名}
   股票代码：{代码}
   交易所：{沪/深/港}

正在扫描全量历史资料...
```

**扫描路径1（Research Vault — 公司目录）：**
`~/Research/Vault_公司基本面Agent/11_公司列表/{拼音}/{公司名}_{代码}/`
→ 年报/、半年报/、季报/、研报/、alphapai/、路演纪要/
→ **纪要对比/** — 历次纪要对比报告
→ **\*研究模型\*.xlsx** — 财务模型（读取/更新时直接在此路径操作）

**扫描路径2（投研报告）：**
`~/0.Agent 投研总监/reports/{公司名}*/`
→ 04_基本面研究.md、05_市场情绪.md、07_风险评估.md、09_综合报告.md

**【重要】扫描历史资料时必须同时扫描两个路径：**
1. `~/Research/Vault_公司基本面Agent/11_公司列表/{拼音}/{公司名}_{代码}/` — 原始资料（年报/季报/历史纪要/研报）
2. `~/0.Agent 投研总监/reports/{公司名}*/` — 投研团队报告（历史判断基准）

财务模型路径（读取/更新）：
`~/Research/Vault_公司基本面Agent/11_公司列表/{拼音}/{公司名}_{代码}/*研究模型*.xlsx`

---

### Step 3：提取纪要关键信息

| 字段 | 来源 |
|------|------|
| 公司对全年收入/毛利率指引 | 纪要原文 |
| 各业务线增速预期 | 纪要原文 |

### Step 4：生成追踪对比报告

**输出路径：**
`~/Research/Vault_公司基本面Agent/11_公司列表/{拼音}/{公司名}_{代码}/纪要对比/`

**报告结构：**
```
## {公司} 纪要追踪对比 — {日期}

### 基本信息
- 纪要来源：{文件名}
- 历史基准报告：{原投研报告日期}
- 对比时间差：{X天}

### 量化指标对比

| 指标 | 历史预测/指引 | 新纪要指引 | 变化方向 | 变化幅度 |
|------|------------|----------|---------|---------|
| 全年收入增速 | XX% | XX% | ↑向好/↓向差 | +Xpp |
| 毛利率 | XX% | XX% | ↑向好/↓向差 | +Xpp |
| ... | ... | ... | ... | ... |

### 定性判断对比

| 维度 | 历史判断 | 新纪要判断 | 变化 |
|------|---------|----------|------|
| 宏观展望 | 谨慎/乐观 | 谨慎/乐观 | ↑改善/↓恶化 |
| 竞争格局 | ... | ... | ... |
| ... | ... | ... | ... |

### 总体评估
- 综合方向：{向好/中性/向差}
- 主要驱动因素：{列举}
- 与历史预期的主要差异：{列举}

### 风险提示
- 哪些历史判断被推翻？
- 哪些新风险点出现？

### 模型调整建议（供确认）
- {指标}：当前预测{X} → 建议调整为{Y}

---
纪要对比完成时间：{YYYY-MM-DD HH:MM}
```

## 文件命名规范

纪要对比文件夹路径：
`~/Research/Vault_公司基本面Agent/11_公司列表/{拼音}/{公司名}_{代码}/纪要对比/`

```
纪要对比/
├── 纪要对比_{纪要日期}.md    ← 本次对比报告
├── 纪要对比_{纪要日期2}.md  ← 历史对比（累计）
├── {公司名}_会议纪要_{日期}.md ← 原始纪要原文
├── 模型更新报告_{日期}.md   ← 模型调整说明（仅选择更新模型时生成）
└── index.md                 ← 追踪记录索引
```

财务模型路径：
`~/Research/Vault_公司基本面Agent/11_公司列表/{拼音}/{公司名}_{代码}/*研究模型*.xlsx`

## 数据源优先级

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1 | 本地文件 | 最新年报/季报/历史纪要 |
| 2 | iFind MCP | `get_stock_financials` 获取最新财务数据 |
| 3 | AlphaPai | 路演纪要、研报摘要 |

---

## 注意事项

1. **自动识别公司**：纪要内容中必须包含公司名称
2. **全量扫描**：两个历史路径都必须扫描
3. **量化优先**：优先提取和对比量化数据
4. **方向明确**：每个对比项必须给出向好/向差/无变化
5. **禁止编造**：财报数据必须来自实际文件，不可用AI生成数据替代
