# 投研追踪对比 Skill — 规划

## Context

用户已有公司历史投研报告，现在有一份新会议纪要，想要：
1. 分析新纪要中的量化数据（管理层指引、业绩预期、关键指标）
2. 与历史投研报告中的判断做前后对比
3. 输出对比报告（向好/向差方向）
4. 由用户决定是否据此更新财务模型 Excel

**适合做成独立 Skill**，与现有"投研团队"互补（投研团队=从0到1，追踪 Skill=从1到N的增量更新）。

---

## 触发方式

触发关键词（与投研团队区分）：
- `追踪 {公司}`
- `纪要对比 {公司}`
- `分析纪要 {公司}`
- `更新追踪 {公司}`

---

## 执行流程（5步）

### Step 1：确认公司 + 读取新纪要 + 全量历史资料扫描

```
输入：{公司名} + {纪要文件路径}
1. 确认股票代码（用iFind或搜索确认）
2. 读取新纪要文件（PDF/DOCX/TXT/MD）

3. 扫描公司历史资料（两个路径都要扫）：
   a. ~/Research/Vault_公司基本面Agent/11_公司列表/{拼音首字母}/{公司名}_{代码}/
      → 扫描：年报/、半年报/、季报/、研报/、alphapai/、路演纪要/
      → 读取最新的基本面资料（年报/季报/历史纪要）
   b. ~/0.Agent 投研总监/reports/{公司名}*/
      → 找到最新一份投研报告（按日期）
      → 读取：04_基本面研究.md、05_市场情绪.md、07_风险评估.md、09_综合报告.md

4. 定位财务模型 Excel（如有）
   → ~/0.Agent 投研总监/reports/{公司名}_{日期}/*.xlsx

输出：建立该公司的"历史资料索引"，供后续步骤按需读取
```

### Step 2：提取纪要关键信息（量化+定性）

从新纪要中提取并结构化：
| 字段 | 来源 |
|------|------|
| 公司对全年收入/毛利率指引 | 纪要原文 |
| 各业务线增速预期 | 纪要原文 |
| 产能/利用率/订单数据 | 纪要原文 |
| 管理层对宏观/行业的判断 | 纪要原文 |
| 任何量化承诺/目标 | 纪要原文 |
| 纪要日期/会议类型 | 文件元信息 |

### Step 3：与历史资料对比

从两个路径读取历史资料：

**路径1 — Research 原始资料：**
- `~/Research/Vault_公司基本面Agent/11_公司列表/{拼音}/{公司名}_{代码}/`
  → 最新年报/季报（量化数据基准）
  → 历史路演纪要（alphapai/ 文件夹）
  → 历史研报摘要

**路径2 — 投研团队报告：**
- `~/0.Agent 投研总监/reports/{公司名}_{日期}/`
  → `04_基本面研究.md` — 历史预测/指引
  → `05_市场情绪.md` — 历史情绪/预期
  → `07_风险评估.md` — 历史风险点
  → `09_综合报告.md` — 历史结论

**对比维度：**

| 对比维度 | 新纪要来源 | 历史基准来源 |
|---------|---------|------------|
| 收入/毛利率指引 | 新纪要 | 04_基本面研究.md + 最新年报 |
| 各业务增速预期 | 新纪要 | 04_基本面研究.md + 最新年报 |
| 宏观/行业判断 | 新纪要 | 01_宏观分析.md（如有） |
| 市场情绪/预期 | 新纪要 | 05_市场情绪.md |
| 风险点 | 新纪要 | 07_风险评估.md |
| 实际财务数据 | 最新季报/年报 | 04_基本面研究.md 预测值 |

**方向判断**：
- 量化指标：实际 vs 指引 → 超预期/符合/低于预期
- 定性判断：更乐观/更谨慎/无变化

### Step 4：生成追踪对比报告

输出到：`~/0.Agent 投研总监/reports/{公司名}_{日期}/追踪对比/`

文件名：`纪要对比_{纪要日期}.md`

**报告结构：**
```
## {公司} 纪要追踪对比 — {日期}

### 基本信息
- 纪要来源：{文件名}
- 历史基准报告：{原投研报告日期}
- 对比时间差：{X天}

### 量化指标对比

| 指标 | 历史预测/指引 | 新纪要指引 | 变化方向 | 变化幅度 |
|------|------------|----------|---------|---------|
| 全年收入增速 | 15% | 18% | ↑ 向好 | +3pp |
| 毛利率 | 42% | 40% | ↓ 向差 | -2pp |
| ... | ... | ... | ... | ... |

### 定性判断对比

| 维度 | 历史判断 | 新纪要判断 | 变化 |
|------|---------|----------|------|
| 宏观展望 | 谨慎 | 乐观 | ↑ 改善 |
| 竞争格局 | 加剧 | 缓和 | ↑ 改善 |
| ... | ... | ... | ... |

### 总体评估
- 综合方向：{向好/中性/向差}
- 主要驱动因素：{列举}
- 与历史预期的主要差异：{列举}

### 风险提示
- 哪些历史判断被推翻？
- 哪些新风险点出现？

### 模型调整建议（供确认）
是否建议调整财务模型：
- {指标}：当前预测{X} → 建议调整为{Y}
- 触发条件：需用户确认后执行

---
纪要对比完成时间：{YYYY-MM-DD HH:MM}
```

### Step 5：等待用户确认是否更新模型

```
报告生成后，展示摘要并询问：
"发现 {N} 项量化指标较历史预测发生变化，是否需要更新财务模型？
  A. 是 — 更新Excel模型
  B. 否 — 保持原模型
  C. 只看报告，不需要更新"
```

选择 A：调用 `RL-company-research-model` 或 `financial-analysis:model-update` 更新 Excel。
选择 B/C：结束。

```
│   ├── 纪要对比_{纪要日期}.md    ← 本次对比报告

│   └── index.md                 ← 追踪记录索引
```

## 数据来源

- **新纪要**：用户提供的文件路径，读取 PDF/DOCX/TXT/MD
- **历史资料路径1（Research）**：`~/Research/Vault_公司基本面Agent/11_公司列表/{拼音}/{公司名}_{代码}/`
  → 年报/、半年报/、季报/（量化基准）、alphapai/（历史纪要）、研报/
- **历史资料路径2（投研报告）**：`~/0.Agent 投研总监/reports/{公司名}_{日期}/`
  → 04~09 各 Agent 报告（判断基准）
- **历史财务模型**：从上述路径2下找同名 .xlsx 文件

## 验证方式

1. 选择一家已有投研报告的公司（如宁德时代）
2. 提供一份模拟会议纪要文件路径
3. 触发"追踪 宁德时代"
4. 检查 `reports/宁德时代_2026-04-06/纪要对比/` 是否生成对比报告
5. 报告中的对比维度是否准确，向好/向差方向判断是否正确
6. 选 A/B/C 选项是否正常响应
