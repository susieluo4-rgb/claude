# Plan: 公司比较 Skill (`equity-research:compare`)

## Context

用户希望创建一个**公司比较** skill，用于横向对比同行业多家公司的财务数据、增长预期和估值水平。输入方式灵活（行业名或代码列表均可），数据源采用 Vault 本地笔记 + ifind MCP 混合模式。

---

## Skill 信息

| 字段 | 内容 |
|------|------|
| **name** | `equity-research:compare` |
| **description** | 输入行业名称或股票代码列表，横向对比同行业多家公司的财务数据、3年增长预期和估值水平。适用于"对比 [行业] 的公司"、"比较 [代码1] 和 [代码2]"、"帮我做同业比较"等场景。输出包含：财务指标对比表、未来3年一致预期、PE/PB/EV-EBITDA 多维估值对比。 |
| **trigger keywords** | 对比、比较、同业比较、公司对比、行业对比 |

---

## 实现方案

### 输入解析
```
用户输入 → 判断是行业名还是代码列表
  ├── 行业名 → 从 Vault 扫描同行业公司笔记 + ifind 补充
  └── 代码列表 → 直接查询这些公司
```

### 数据获取流程（ifind 实时优先）

```
用户输入公司列表
  ↓
始终优先调用 ifind MCP 实时查询（即使 Vault 有笔记也查）
  ├── get_stock_info         → 基本信息、行业分类
  ├── get_stock_financials   → 收入、毛利、EBIT、净利润、ROE
  ├── get_stock_performance  → 市值、PE、PB、EV-EBITDA
  └── get_risk_indicators    → Beta、波动率
  ↓
Vault 笔记仅在以下情况降级读取：
  ├── ifind 查询失败时
  └── 历史研究背景补充（可选，读最新日期的笔记）
```

> **数据优先级**：始终以 ifind 实时数据为准，Vault 仅作 ifind 不可用时的备用。

### 输出格式：Markdown 对比报告 + Excel 文件

**Markdown**：直接在对话中输出，供快速浏览。
**Excel 文件**：保存到 Vault `01_公司研究/` 目录，文件名格式：
```
公司比较_[行业/主题]_[YYYYMMDD].xlsx
```
Excel 包含多个 sheet：财务数据、预期增长、估值对比、原始数据（ifind 返回的原始字段）
