## 交互式工作流程

每次触发后，Lead Agent 通过**菜单选项**引导用户推进任务。

---

### 第一步：确认任务（含建模选项）

```
🎯 投研任务已启动

目标公司/行业：{用户输入}
任务类型：全量投研

请选择是否需要建立财务模型：
  A. 【建立模型】— 触发 RL-company-research-model，生成完整三表+估值 Excel
  B. 【不做模型】— 仅输出文字报告，跳过财务建模
  C. 查看当前已有报告
  D. 取消
```

**"建立模型"选项说明**：基本面Agent完成后，自动调用 `RL-company-research-model` Skill，根据 CATL 模板生成含三张表、业务拆分、估值sheet的 Excel 模型。结果文件路径传递给基金经理Agent使用。

**"不做模型"选项说明**：跳过建模流程，直接进入后续Agent，输出纯文字投研报告。

### 第一步附：雷达Agent数据预收集

雷达Agent是投研系统的**前置数据收集层**，在其他Agent启动前预先抓取目标公司的外部数据（财报/研报/纪要/宏观/行业）。

```

数据收集范围：
  📊 财务数据（iFind）
  📋 公司信息（iFind）
  📢 公告列表（iFind）
  🎤 路演纪要（AlphaPai）
  📄 研报摘要（AlphaPai）
  🌍 宏观/行业数据（iFind EDB）
  📁 本地已有文件扫描

请选择：
  A. 继续等待完成
  B. 查看已收集数据
  C. 跳过，直接触发第一批Agent（使用已有数据）
  D. 取消任务
```

雷达Agent完成后自动继续至第二步。

## 报告输出规则（强制执行）

每次投研完成后，**必须**执行以下输出操作：

### 输出文件
| 文件类型 | 保存路径 | 说明 |
|---------|---------|------|
| **Markdown源报告** | `/Users/zhuang225/0.Agent 投研总监/reports/{公司名}_{日期}/` | 每个Agent生成独立MD文件（00-09编号） |
| **交互式HTML报告** | `/Users/zhuang225/0.Agent 投研总监/reports/{公司名}_{日期}/index.html` | 整合所有MD报告为可搜索、可打印的HTML |

### HTML报告要求
- **侧边栏导航**：各Agent报告页面（综合、宏观、行业、数据校验、基本面、情绪、技术、风险、基金经理）
- **搜索功能**：支持关键词高亮
- **打印按钮**：导出PDF
- **KPI卡片**：核心数据一目了然
- **颜色标签**：风险等级（绿/黄/红）
- **催化剂时间轴**：短期/中期/长期

### HTML合成（强制）
**必须调用 `RL-html-report-builder` Skill** 生成 HTML 报告，禁止手动拼凑 HTML。具体规范参见该 Skill，核心要点：
- 表格正则必须使用 `r'(\|.+\|\n\|[-| :]+\|(?:\n\|[^\n|]+\|)+)'`（`[^\n|]+` 替代 `.+`，避免贪婪匹配导致多行数据丢失）
- KPI 网格 `minmax(155px,1fr)` + `font-size:12px`（防止中文标签挤压）
- CSS 颜色值不得出现双 `#` 符号（如 `##c5cae9` → `#c5cae9`）
- 构建脚本命名：`build_{公司名}_html.py`，存放于 `reports/` 目录

### 文件命名规范
```
{公司名}_{YYYY-MM-DD}/
├── 00_综合报告.md
├── 01_宏观环境.md
├── 02_行业分析.md
├── 03_数据校验.md
├── 04_基本面研究.md
├── 05_市场情绪.md
├── 06_技术分析.md
├── 07_风险评估.md
├── 08_基金经理评估.md
├── 09_综合报告.md
└── index.html
```

## 基本面Agent前置资料核查规则（强制执行）

基本面Agent在开始分析前，**必须**先完成资料核查与补全，不得跳过。

### 核查清单
| 资料类型 | 要求 | 存放路径 |
|---------|------|---------|
| 年报 | 近两年（2024+2025） | `11_公司列表/{拼音首字母}/{公司}_{代码}/年报/` |
| 半年报 | 近两年（2024+2025） | `11_公司列表/{拼音首字母}/{公司}_{代码}/半年报/` |
| 季报 | 近四期（覆盖4个季度） | `11_公司列表/{拼音首字母}/{公司}_{代码}/季报/` |
| 研报 | 券商研报、公司介绍等 | `11_公司列表/{拼音首字母}/{公司}_{代码}/研报/` |

### 执行流程
```
Step 1: 检查文件夹是否存在
  → 11_公司列表/{拼音首字母}/{公司名}_{代码}/ 是否存在
  → 不存在则创建：年报/、半年报/、季报/、研报/、会议纪要/

Step 2: 检查资料完整性
  → 年报：2024年、2025年是否齐全
  → 半年报：2024年、2025年是否齐全
  → 季报：是否有近四期
  → 研报：是否有基础研报

Step 3: 补全缺失资料（按优先级）
  1. 本地Vault文件夹
  2. iFind API (search_news/search_notice)
  3. 网络搜索官方财报链接

Step 4: 请求用户协助（最后手段）
  → 仅当所有自动途径均失败时
  → 询问话术："缺少{公司}的{资料类型}，是否需要您手动协助下载？"

【绝对禁止】
- 不得自行编写/编造任何财报数据
- 不得使用过期数据冒充最新数据
- 不得在资料缺失时强行开始分析
```

## 财报数据优先级规则（所有基本面分析适用）

**【强制规则】分析任一公司时，必须按以下顺序查找并使用最新财报数据：**

### 数据源优先级
| 优先级 | 来源 | 说明 |
|--------|------|------|
| **1. 本地文件** | `Vault_公司基本面Agent/11_公司列表/{拼音首字母}/{公司名}/` | 优先查找最新年报/半年报/季报PDF |
| **2. iFind MCP** | `get_stock_financials` | **必须使用自然语言query格式**：`get_stock_financials({ query: "{公司名} {代码} {年份} {指标}" })` |
| **3. 网络搜索** | `search_news` + 网页抓取 | 查找最新财报公告、业绩预告 |

### 执行流程
```
Step 1: 搜索本地目录
  → Glob "**/{公司名}*/年*" 查找最新年报
  → Glob "**/{公司名}*/季*" 查找最新季报
  → 如找到2025年年报，必须使用2025年数据，不得使用2024年

Step 2: iFind API 校验
  → 调用 get_stock_financials 确认最新一期财务数据
  → 对比本地文件与API数据一致性

Step 3: 如本地无最新财报
  → Agent上网搜索" {公司名} 2025年年报"或" {公司名} 业绩预告"
  → 找到后下载/读取，补充至本地Vault目录
```

### 错误示例（禁止）
- ❌ "2024年年报显示营收..."（实际2025年年报已发布）
- ❌ "根据2024年三季报..."（实际有2025年三季报）
- ❌ 使用陈旧数据导致盈利预测与最新公告矛盾

### 正确示例
- ✅ "根据2025年年报，最新营收为4237亿元..."（已同步最新数据）
- ✅ "2025年一季报净利润同比+42.3%，大超预期"（使用最新季报）
